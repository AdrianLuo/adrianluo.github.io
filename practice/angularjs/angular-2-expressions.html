AngularJS Expressions vs. JS Expressions

like JS expressions, AngularJS expressions can contain literals, operators, and variables

Unlike JS expressions, AngularJS expressions can be written inside HTML

AngularJS expressions do not support conditionals, loops, and exceptions, while JS expressions do

AngularJS expressions support filters, while JS expressions do not

AngularJS Expression

Expressions are written inside double braces { { expression } }

Expressions can also be written inside a directive: ng-bind="expression"

AngularJS will resolve the expression, and return the result exactly where the expression is written

AngularJS expressions are much like JavaScript Expressions: they contain literals, operators and variables { { 5 + 5 } } or { { firstName + " " + lastName } }

My expression: 5 + 5 = {{ 5 + 5 }}

Example of changing css properties with an initialized variable: ng-init="myCol='#888'"

AngularJS Numbers

AngularJS numbers are like JS numbers

Example - Init: ng-init="quantity=2;cost=3", expression: { { quantity * cost } }, binding: ng-bind="quantity * cost"

Using expression: Total in dollars {{ quantity * cost }}

Using ng-bind: Total in dollars

AngularJS Strings

AngularJS strings are like JS strings

Example - init: ng-init="firstName='John';lastName='Doe'", expression: { { firstName + " " + lastName } }, binding: ng-bind="firstName + ' ' + lastName"

Using expression: The name is {{firstName + " " + lastName}}

Using ng-bind: The name is

AngularJS objects are like JS objects

Example - init: ng-init="person={firstName:'John',lastName:'Doe'}", expression: { { person.lastName } }, binding: ng-bind="person.lastName"

Using expression: The name is {{person.lastName}}
